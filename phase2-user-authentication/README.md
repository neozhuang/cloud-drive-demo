# Cloud Drive Demo - Phase 2 User Authentication & Performance Optimization

在Phase 1基础框架上的增强版本，添加了用户认证系统、完善的日志记录、大文件传输优化和断点续传功能。

## 主要改进

### 1. 用户认证系统
- **新增LOGIN命令类型**：在`packet.h`中添加了`LOGIN`和`CLIENT_EXIT`命令
- **基于系统用户的认证**：使用`getspnam()`和`crypt()`进行用户名/密码验证
- **用户隔离目录**：每个认证成功的用户拥有独立的云盘目录
- **会话状态增强**：`session_t`结构添加了`uid`、`gid`、`is_authenticated`等字段

### 2. 完善的日志系统
- **结构化日志记录**：在`server/include/log.h`中定义了完整的日志系统
- **多级别日志**：支持`LOG_LEVEL_INFO`、`LOG_LEVEL_WARN`、`LOG_LEVEL_ERROR`等
- **详细事件记录**：
  - 连接/断开连接事件
  - 用户认证事件（成功/失败）
  - 文件传输事件（上传/下载，包含大小、时长、状态）
  - 命令请求事件

### 3. 大文件传输优化（mmap）
- **智能阈值判断**：超过100MB的文件自动使用mmap优化
- **双向mmap支持**：
  - 客户端上传大文件：使用mmap映射文件并分块发送
  - 服务器接收大文件：使用mmap映射目标文件直接接收
  - 服务器发送大文件：使用mmap映射源文件并分块发送
  - 客户端下载大文件：使用mmap映射目标文件直接写入
- **进度显示**：每传输10MB显示一次进度百分比

### 4. 断点续传功能
- **PUTS断点续传**：
  - 客户端发送文件前先获取服务端已存在文件的大小
  - 根据本地偏移继续上传剩余部分
  - 服务端支持追加模式打开文件
- **GETS断点续传**：
  - 客户端下载前发送本地已存在文件的大小
  - 服务器从相应偏移开始发送数据
  - 客户端支持追加模式写入文件

### 5. 增强的错误处理和监控
- **详细的传输统计**：记录每次文件传输的开始/结束时间、传输大小、时长
- **传输失败分类**：区分文件不存在、权限错误、网络错误等不同类型的失败
- **状态码细化**：用户认证返回0（成功）、1（用户不存在）、2（密码错误）

## 使用方法

### 1. 启动服务端
```bash
cd server
./server
```
服务端会从 `server.conf` 读取配置（默认：`127.0.0.1:8888`，4个工作线程）。  
你也可以指定自定义配置文件：
```bash
./server /path/to/config.conf
```
或者直接提供参数：
```bash
./server <server_ip> <server_port> <thread_count>
```

### 2. 运行客户端
```bash
cd client
./client <server_ip> <server_port>
```
示例：
```bash
./client 127.0.0.1 8888
```

连接成功后，客户端会提示输入用户名和密码进行认证。

### 3. 可用命令
| 命令 | 描述 |
|---------|-------------|
| `pwd`   | 显示当前工作目录 |
| `cd <目录>` | 切换目录 |
| `ls [路径]` | 列出目录内容（简短格式） |
| `ll [路径]` | 列出目录内容并显示详细信息（权限、大小、时间） |

## 协议增强

### 用户认证流程
1. 客户端连接后发送`LOGIN`命令包，分别包含用户名和密码
2. 服务端接收用户名，密码
4. 服务端验证用户名和密码
5. 服务端返回认证结果（0=成功，1=用户不存在，2=密码错误）

### 文件传输协议增强
#### 断点续传流程（PUTS）
1. 客户端发送`PUTS`命令，包含文件名
2. 服务端检查目标文件大小，返回已存在大小
3. 客户端发送文件总大小
4. 如果已存在大小小于总大小，客户端从偏移处开始发送
5. 服务端以追加模式接收剩余数据

#### 断点续传流程（GETS）
1. 客户端发送`GETS`命令，包含文件名
2. 服务端返回文件总大小
3. 客户端发送本地已存在文件大小
4. 如果已存在大小小于总大小，服务端从偏移处开始发送
5. 客户端以追加模式接收剩余数据

## 性能优化

### mmap大文件传输
- **阈值**：100MB（`MMAP_THRESHOLD`）
- **优势**：
  - 减少系统调用：从数万次`read()`/`write()`调用减少到几次`mmap()`/`munmap()`调用
  - 零拷贝优势：数据直接从映射内存区域传输，避免额外缓冲区复制
  - 内存效率：操作系统自动处理页面缓存和脏页回写
- **实现**：在`transmit_data.c`中同时实现了客户端和服务端的mmap优化

## 已知限制

- 用户认证依赖于系统用户账户（`/etc/shadow`）
- 用户密码采用明文传输
- 不支持用户注册功能，需要预先在系统中创建用户
- mmap优化需要足够的虚拟内存空间
- 未对传输文件进行哈希校验，确保传输内容的正确性
